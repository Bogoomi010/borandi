# 선택 유닛 상세 카드 — UI 템플릿 제작 계획

> 대상 컴포넌트: `#unit-detail` (단일 유닛 선택 시) · 렌더 함수 `renderUnitDetail()` (`src/ui/panels.ts:463`)

> 테마: 차원 균열 랜덤 디펜스 / 다크 블루+골드 마법 HUD, 픽셀아트

## 1. 결론 먼저 (이 카드의 렌더링 구조)

이 카드는 **캔버스에 스프라이트를 찍는 방식이 아니라, DOM 요소에 PNG를 입히는 방식**이다. 두 가지 기법만 쓴다.

1. **9-slice 프레임** — `applyNineSlice(el, key, { slice, width, fill })` → CSS `border-image`. 모서리 장식은 고정, 가운데/모서리는 늘어남(stretch). 그래서 **한 장의 PNG로 가변 크기 박스를 다 커버**한다. 카드의 모든 프레임(패널/포트레이트/스탯격자/슬롯)이 여기 해당.
2. **배지/아이콘 배경** — `applySpriteLocalized(el, key)` → `background-image` (`contain`). 늘어나지 않는 고정 비율 이미지. 공격타입 배지가 여기 해당하며 **언어별(`_ko/_en/_jp/_cn`)** 파일을 자동 선택한다.

추가로 아이콘(`공격력/속도/사거리/타겟/누적딜` 등 작은 아이콘)은 **단색 SVG 마스크**라서 CSS `color`로 등급색·속성색에 맞춰 자동 틴팅된다 → **픽셀아트로 새로 뽑을 필요 없음**(원하면 교체 가능, 후순위).

핵심 시사점:
- 9-slice 프레임은 **정확한 픽셀 크기에 구애받지 않는다.** 중요한 건 ① 모서리 장식이 `slice` px 안에 들어올 것 ② 가로/세로로 늘어나도 안 깨지는 무늬일 것.
- 따라서 GPT로 뽑을 때 **"정사각/적당히 큰 캔버스 + 모서리 장식 + 늘어나도 되는 중앙"** 으로 만들고, 코드의 `slice` 값에 맞춰 모서리 두께만 지키면 된다.

## 4. 상태·동작 시 추가로 필요한 이미지 (변형 매트릭스)

카드는 대체로 정적이지만, **상태에 따라 색/프레임/오버레이가 바뀐다.** 무엇을 "CSS 틴팅으로 처리"하고 무엇을 "별도 이미지로 구워야" 하는지가 제작량을 좌우한다.

### (A) 등급(Grade) — 5종 `일반/희귀/영웅/전설/히든`
- 현재: `--grade-color` CSS 변수로 카드 상단 보더·포트레이트 테두리·아이콘색을 **자동 틴팅**.
- **권장: 프레임은 1장(중립 회색조)으로 만들고 색은 CSS에 맡긴다.** → 등급별 프레임 5장 불필요.
- 단, 등급 엠블럼(`grade-frame`)을 "구운 배지"로 쓰려면 **5변형 필요**(또는 별 1~5개를 별 스프라이트로 조합).

### (B) 속성/계열(Family) — 6종 `화염/서리/폭풍/강철/공허/숲`
- 현재: `--unit-color`로 포트레이트 배경 그라데이션 자동 틴팅.
- 프레임 자체는 속성 무관 1장으로 충분. **속성 차이는 캐릭터 픽셀아트(§5)와 색으로 표현.**

### (C) 공격 타입 배지 — `physical/magic/pierce/true` × 언어 4
- 현재 art 존재: magic, physical (각 `_ko/_en/_jp/_cn`).
- **부족분: `pierce`(관통), `true`(고정) 2종 × 4언어 = 8장** 추가하면 텍스트 폴백 제거 가능.

### (D) 슬롯 상태 — 동작 중 변함
- `slots.skill`(활성) / `slots.empty`(빈) / `slots.locked`(잠금) — 3상태 = 3장 (있음).
- **`slots.cooldown-overlay`**(240×226 있음): 전투 중 스킬 쿨다운 표시용 **반투명 스윕 오버레이**. 활성 슬롯 위에 올려 회전/감소 애니메이션. → 동작용 추가 이미지 ✔
- 선택 시 테두리 연출: `units.selected-card-frame`(애니메이션 테두리). 카드/슬롯 선택 글로우. → 동작용 ✔

### (E) 다중 선택 모드
`#unit-detail.multi` (2기 이상 선택). 포트레이트가 "선택 N기" 요약으로 바뀜. 프레임은 동일 템플릿 재사용, **추가 이미지 불필요**(텍스트만 교체).

### 정리: 동작 시 추가가 필요한 이미지
1. **쿨다운 오버레이** (슬롯용) — 이미 placeholder 있음, 픽셀 톤으로 재제작 권장
2. **선택 글로우 프레임** (애니 테두리) — placeholder 있음
3. **공격타입 pierce/true 배지** × 4언어 — 신규
4. (선택) 등급 엠블럼 5변형 — "구운 배지" 채택 시에만

## 5. 캐릭터 픽셀아트 (포트레이트) — 템플릿이 아니라 유닛별 에셋

초상화 프레임(#2)은 **빈 금색 테두리**이고, 그 안의 캐릭터는 유닛마다 1장씩 별도다 (`renderUnitDetail` 주석: "추후 유닛 캐릭터 이미지를 넣으면 됨"). 현재는 이니셜+색 임시.

- **포트레이트 권장 캔버스: 200×200 (@2x, 투명배경)** — 프레임 안쪽 ~88×100 표시 영역에 들어가도록 여백 포함.
- 전투 보드용 풀 애니메이션은 별개(이미 `unit-legendary-forest/animation-*` 에 idle/walk/attack/skill/death × east/west/south 스프라이트 구조 존재). **카드 HUD는 정지 1컷이면 충분.**
- 6속성 × 등급별 유닛 수만큼 필요 → GPT 일괄 생성 대상. 프레임/슬롯 같은 "템플릿"과 분리해서 관리.

## 6. GPT 프롬프트용 제작 스펙 (공통 규칙)

모든 템플릿 공통:
- **투명 배경 PNG**, 픽셀아트, 다크 블루(#0d1016~#222834) + 글로우 블루(#3f86e6) + 골드(#e8a33d) 팔레트.
- **9-slice 프레임(#1,2,4,5,6,7,8)**: 가운데를 비우거나 단순 타일 무늬로. 장식은 **네 모서리에 집중**, 모서리 장식이 표의 "코너 px" 안에 들어오게. 상/하/좌/우 변은 좌우·상하로 늘어나도 자연스러운 반복 무늬.
- **배지(#9,10,11)**: 비늘림. 글자 새겨 넣을 땐 언어별 4장 세트. 가운데 정렬, 여백 균등.
- 파일명 규칙: 언어 변형은 `<base>_ko.png / _en / _jp / _cn`. 코드가 자동 폴백.
- 저장 위치: `src/assets/ui/unit-detail/`, 슬롯은 `src/assets/ui/slots/`.

권장 제작 우선순위:
1. 포트레이트 프레임(#2) · 스탯격자(#4) · 슬롯 3종(#6,7,8) — **라이브 카드 즉시 반영**
2. 이름 배너(#3) · 패널 배경(#1)
3. 공격타입 pierce/true(#9,10 확장) · 쿨다운/선택 글로우(동작)
4. 캐릭터 포트레이트(유닛별) · 등급 엠블럼(선택)
